現代前端新手的 Next.js 建立靜態網站初體驗

Authors
  • Name
    kevin wang
    Twitter

前言

在 2020 年時我曾經用 GastbyJS 搭配 Starter 專案修改出一個部落格，
寫了幾篇文章因為忙碌的關係大約快一年沒有寫文章了。
不知不覺到了 2022 年 6 月，
一直覺得將自我進修的成果整理後分享可以更增進自己的理解，
於是興起了繼續寫部落格文章的念頭。

一直覺的第一次透過靜態網站產生器套用 Starter 專案修改出的部落格很陽春，
本來想找簡單且穩定有更新的 Starter 專案持續客製化，
但其實從架好部落格到現在都沒時間做客製化(甚至停更了一年...)，
其實 GastbyJS 的有不少不錯的 Blog Starter 專案，
但足夠開箱即用的專案的 Star 數都偏少，
看來看去 tailwind-nextjs-starter-blog 這個 Starter 專案的 Star 相當多且有贊助者，
而其完整度足夠我開箱及用。
所以我便決定將部落格遷移到這個專案。

開箱

在 README.md 有提到操作，以下用中文簡單帶過。

首先要執行以下指令裝好 node_module

npm install

再來透過以下指令啟動開發伺服器

npm start

我一開始在啟動開發伺服器時我有碰到啟動失敗的情形，
後來在升級 Node.js 版本 (12 lts -> 16 lts) 後就正常了。

需要注意的地方

Starter 專案內的 README.md 詳細的講解了客製化該注意的地方，

底下列出我有修改的部分

data/siteMetadata.js

這個檔案內記錄了這個網站的使用者可修改相關資訊，
網站要套用專案時務必來改這個檔案。

電子報跟留言的功能也是在這個檔案內設定，
如果沒有在使用的話 newsletter.provider 以及 comment.provider 放空字串就可以了。

在 commit 檔案的時候有看到一些警告訊息，
如果是用 Windows 時會碰到的第一個問題是提示了以下警告

npm WARN config global `--global`, `--local` are deprecated. Use `--location=global` instead.

參考了 npm github issue 的說明， 修改了 npm.cmd 後查看 npm -v 時警告就消失了。

再來一個警告是

Browserslist: caniuse-lite is outdated. Please run:
npx browserslist@latest --update-db
  Why you should do it regularly: https://github.com/browserslist/browserslist#browsers-data-updating

按照警告的提示執行 npx browserslist@latest --update-db 後再次 commit 這個警告就消失了，

但還是提示剛剛出現過的警告

npm WARN config global `--global`, `--local` are deprecated. Use `--location=global` instead.

參考了 npm github issue 的說明，

執行了以下指令

npm install --global --production npm-windows-upgrade
npm-windows-upgrade --npm-version latest

再次 Commit 就沒有警告了。

如果使用系統管理員權限的 PowerShell 視窗依然不能執行上面指令的話，
可以透過以下指令暫時的提高當前 PowerShell 的執行原則

Set-ExecutionPolicy Bypass -Scope Process

如此一來 Commit 時出現的警告就清空了。

data/authors/default.md

這個檔案是預設的作者介紹，把 md 檔內容做一點修改就可以了。

data/projectsData.js

這個檔案是介紹自己作品的資料，修改裡面的陣列資料就好。

data/headerNavLinks.js

這個檔案是網站上方的導覽列，可以透過修改陣列資料改變導覽列內容。

data/logo.svg

網站的 logo 圖

data/blog

文章都會放在這裡，同時支援 md 以及 mdx，可以看專案內的範例文章快速了解一下有支援的用法。

public/static

這個位置都是放一些靜態檔案，像是圖片等，可以把專案內的用不到的範例圖刪除。

layouts

這個資料夾內都是放頁面會用到的主要樣板，如果像我一樣沒有提供 Twitter 的話就可以在專案中尋找 Discuss on Twitter 字串找到 PostLayout.js，註解掉相關元素就不會在文章內顯示了。

發布網站

參考 Next.js 官方文件，我們需要將 package.json 內的 build 指令更換成

next build && node ./scripts/generate-sitemap && next export

就可以呼叫 npm run build 做發布了。

發布後會發生錯誤是正常的，參考錯誤訊息會發現提示 next export 無法正確處理 Image Optimization，在 Github 上的 READ.md 有具體的解釋。
文件提到如果要發布成靜態網頁的話，需要將 next/image 元件換成標準的 <img> tag。
另一個思路則是更換 cutom loader，
參考 Next.js 在 Github 的討論串，
調整 Image.js 的程式碼，

import NextImage from 'next/image'

const customLoader = ({ src }) => {
  return src
}

export default function Image(props) {
  return <NextImage {...props} loader={customLoader} />
}

透過直接 bypass src 的方法可以忽略掉圖片優化。

目前沒有看到完美的開啟圖片優化方案，
這塊我打算後續再研究研究。

到此發布網站的準備就完成了，成功發布後產出會在 out 資料夾。
可以透過 http-server 套件快速啟動一個靜態網頁的伺服器查看發布的結果。

直接在 out 資料夾cd 執行

npx http-server .

或是全域安裝後再執行

npm install --global http-server

將網頁發布到 GitHub Page

GitHub Page 搭配 GitHub Action 非常方便，可以將 Starter 專案透過 GitHub 提供的機器發布到公開專案並以 GitHub Page 的形式顯示。
需要以下步驟來完成發布

建立 GitHub Page 使用的 Repo

在 GitHub 建立一個 username.github.io 的 Repo

產生 Personal access token

到 GitHub 設定頁產生 Token 並記錄起來。 如果 Starter 專案是 Public Repo 的話選 public_repo，
如果 Starter 專案是 Private Repo 的話要選整個 Repo 權限。

設定 Secrets

在 Starter 專案 Repo 的 settings/secrets/actions 頁建立一組 secret

新增 GitHub Action yaml

Gihub 官方文件 對 GitHub action 有很多詳細的介紹。

以下是我所時做過的一些紀錄

在 .github 資料夾內新增 workflows/{workflow-name}.yaml

參考以下設定

name: Publish
# 在 master 有 push 時觸發動作
on:
  push:
    branches:
      - master

jobs:
  # 這個是 job_id
  build:
    # 指定這個 job 要跑在什麼環境上 要注意 GitHub Runner 每次都會是乾淨環境
    # 可以指定 self-hosted runner 然後宣告成
    # [self-hosted, windows/linux/macOS,x86/x64/ARM/ARM64]
    runs-on: ubuntu-latest
    # 每個 job 會有很多的任務，這些任務集合叫做 steps，會依序執行
    steps:
      # uses 代表套用現成的action
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.剛剛設定的SecretName }}
          # 發佈到 GitHub Page 所在 Repo
          external_repository: `{username}`/`{username}`.github.io
          publish_branch: master
          publish_dir: ./public

都設定正確的話， 當 master 有新的 push 的時候， 就會自動執行設定好的 GitHub Action。

GitHub Action 的計費方式

可以在 Github 帳單頁查看帳單， 如果是在 Public Repo 執行 Action 的話是不用錢的， 而在 Private Repo 則有額度的限制。
