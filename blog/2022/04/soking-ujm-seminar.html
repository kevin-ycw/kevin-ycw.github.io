聽完 Soking UJM 講座後的筆記

  • Name
    kevin wang

前言

這是今年四月聽完 UJM 講座的筆記。
聽完這個講座後，我整理了一些筆記，並在公司實際帶了一場 UJM Workshop，也將 UJM 用在了工作上的需求釐清上。

要怎麼描述產品服務如何與用戶發生關係

Q: 如何評估新市場用戶
A: 投廣告啊! 看數據!
=>已轉換數據是後設指標，他是轉換過的"結果"

以下哪個敘述是用戶旅程

  1. 展示用戶如何在介面之中移動的流程 => 資訊架構(Information Architecture)、sitemap

  2. 展示用戶與產品服務互動關係的發展過程 => 用戶旅程

  3. 展示產品服務如何運作的任務流程 => user flow(用戶在邏輯上會經過的行為任務)

用戶旅程圖的常見誤區

  1. 變成服務流程的討論
  2. 成為創意發想工具(e.g: event storming) => 基於假設的憑空發想且沒有觀察現實用戶 => 變成利害關係人許願池

什麼是用戶旅程圖

用戶旅程圖就是 人們在一段時間範圍內，對研究題目從產生興趣到產生行為的過程，在過程中會有:

  • 服務接觸點（Touch Point） ：旅程中的重要關鍵時刻 MOT（Moments of Truth）是什麼？
  • 時間點 / 旅程階段（Stage）：整個旅程進行的順序？可分為哪些階段？
  • 痛點 / 甜蜜點（Think Purpose）：旅程中的互動，對什麼感到不足或是感到滿足？
  • 情緒（Emotional Journey）：在旅程中的每一個階段，所產生的情緒變化？

用戶旅程圖的前置工作-人物誌

每個用戶旅程圖，都需要一個主角。
用戶旅程圖的用戶，我們可以透過 人物誌 進行描述。

人物誌的執行過程

  1. 執行用戶訪談
  2. 分析行為並標籤化
  3. 標籤分類

Q: 為什麼要建立人物誌
A:

  1. 具體交付: 一個對用戶研究的具體紀錄
  2. 凝聚共識:清楚描述用戶形象，可以有案例來說明用戶處境及需求
  3. 客觀抽離: 避免關連到現實角色產生主觀意識

用戶旅程圖常見元素

  1. 用戶目標與痛點(人物誌上會有)
  2. 用戶行為
  3. 用戶行為階段 => 分類用戶行為的階段
  4. 活動、工具、媒介(同理心地圖上會有)
  5. 用戶情緒點 =>人類的決策其實是從情緒推動的，理性與邏輯隨之在後
  6. 關鍵時刻接觸點 =>人類能記住的東西通常跟情緒有關

從接觸點的用戶痛點與機會找到產品價值

服務接觸點的觀察指標

  1. 引流
  2. 轉換
  3. 重複
  4. 傳播

拆解關鍵時刻的三元素

傳播

  1. 是誰 => 划手機的媽媽
  2. 在什麼狀況下 => 看到許多網紅推薦同一個韓國氣炸烤箱
  3. 感受到什麼 => 氣炸鍋社團的網友討論材質，覺得有質感

引流

  1. 是誰 => 工作繁忙的老爸
  2. 在什麼情況 => 家裡的氣炸鍋壞很久了一直沒有買
  3. 感受到什麼 => 老婆似乎有中意的品牌了

轉換

  1. 是誰 => 擔憂小孩吃不健康零食的爸爸
  2. 在什麼情況 => 正在猶豫規格該不該買，比較規格中
  3. 感受到什麼 => 韓國氣炸烤箱，居然可以烘烤果乾

重複

  1. 是誰 => 遠距工作者爸爸
  2. 在什麼情況 => 早上起床工作時幫自己準備早餐
  3. 感受到什麼 => 使用後清潔很方便

四種關鍵時刻接觸點對應哪類問題?

A. 為什麼有的用戶很熱情地到處推薦我們產品，但是有的用戶只有默默使用?
誰才是對的族群?

=> 傳播

B. 為什麼有的用戶只轉換一次就再也不回頭了? 各種方式都無法再次喚回?
到底問題在哪裡?

=> 重複

C. 為什麼有的用戶總是進來看看但死都不轉換?
為什麼有的用戶一看就買?
難道我做對了什麼?

=> 轉換

D. 為什麼有的用戶一看到我們的產品服務立刻就感興趣?
但有些用戶看都不看一眼?

=> 引流

總結

  1. 用戶旅程圖展示用戶與產品互動關係的發展過程
  2. 先定義用戶旅程中的主角: 人物誌
  3. 透過用戶旅程圖了解用戶歷程與接觸點

三種不同樣貌的用戶旅程

1. 用戶行為的旅程，新用戶

探索用戶旅程用於尋找新用戶接觸點

  • 情境分析
    訪談利害關係人，確認產品服務目前定義的核心使用情境

  • 用戶訪談
    針對目標族群進行招募，進行深度訪談，了解使用情境的痛點與期望

  • 用戶旅程視覺化
    將使用情境的歷程進行視覺化展示，轉換成卡片流程或是工作坊討論

  • 辨識接觸點
    與利害關係人一同分析旅程中識別出來的接觸點，討論產品需求範圍

探索用戶行為的旅程就是
    了解用戶與我們相遇的接觸點
    用以設計最適合的迎接姿態

2. 會員服務的旅程，新產品

探索會員服務旅程用於開發新服務

  • 脈絡訪查
    訪談利害關係人與觀察現場的運作，確認現行流程的整理脈絡

  • 用戶訪談
    針對目標族群進行招募，進行深度訪談，了解使用情境的痛點與期望

  • 用戶旅程視覺化
    將使用情境的歷程進行視覺化展示，轉換成卡片流程或是工作坊討論

  • 辨識接觸點
    與利害關係人一同分析旅程中識別出來的接觸點，討論產品需求範圍

探索會員服務的旅程就是
    探索用戶與我們服務接觸的關鍵時刻
    用以洞察體驗過程的峰值感受

3. 專案開發的旅程，新流程

透過用戶旅程工作坊進行團隊共識討論

  • 需求訪談
    訪談利害關係人，確保決策者有改變的意願，以及事先了解多重觀點

  • 設定議題
    從利害關係人訪談中，尋找議題關鍵字，以及議題上引發的現象

  • 挑選成員
    關心核心議題影響範圍的上下游部門成員，對議題感同身受

引導流程一:議題與目標

  1. 介紹議題
    說明核心議題，設定工作坊討論範圍，確保參與者感受自己的價值
  2. 描述目標
    分組討論定義出理想終點以及具體條件，減少想像的歧義

引導流程二: 任務與困境

  1. 描述任務
    邀請跨部門參與者，共同產出任務描述卡片，探索流程的起點如何抵達終點
  2. 描述困境
    共同發想補充，描述不同任務階段觀察到的問題

團隊共識之後的卡片分類討論

以用戶旅程討論團隊困境的三個好處

  • 節省認知資源
    議題視覺化，將整體問題的脈絡展現在眼前，可觀察整體也可討論單點
  • 控制議題範圍
    旅程圖有明確的共同目標終點可以確保聚焦，也有足夠的發散空間可供稿論
  • 心理安全空間
    將議題變成外部的卡片，避免過度與發言者的自尊心連結，創造安全感
探索專案的旅程就是
    團隊成員**共同意識**期待與現實的落差
    用以討論行為如何改變的旅程

總結一下

  1. 旅程圖可以挖掘新市場、新用戶的 接觸點
  2. 洞察會員旅程協助數位轉型 釐清需求情境
  3. 以 團隊共識討論法 建構旅程圖協助流程改善
